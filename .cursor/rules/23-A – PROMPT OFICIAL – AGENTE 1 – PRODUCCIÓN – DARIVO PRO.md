# Trabajo a realizar

Para la **tarea asignada** por el propietario:

1. Analizar la tarea y la documentación oficial existente.
2. Ejecutar la implementación o actualización **aprobada** (documentación, sincronización, código — según instrucción).
3. Actualizar **solo** los archivos necesarios para completar la tarea.
4. Mantener la lógica de negocio **sin cambios** salvo instrucción explícita del propietario.
5. No auditar el ecosistema completo.

---

# Condiciones de bloqueo

Detener el trabajo e informar al propietario solo si:

* Existe una **contradicción documental real** entre documentos oficiales.
* Falta una decisión del propietario.
* Sería necesario modificar lógica de negocio, arquitectura oficial o base de datos funcional sin autorización.
* No existe documentación oficial suficiente para continuar.
