# Cambios 21/07/2026 — Instalación de Latitude Telemetry (observabilidad de las llamadas a OpenAI)

Pedido explícito de instalar la skill `latitude-telemetry` (`github.com/latitude-dev/skills`) y usarla para instrumentar la app. Skill instalada en `.claude/skills/latitude-telemetry/SKILL.md` (solo ese archivo existe en el repo de origen). Ejecutado su propio flujo (auditoría → agrupar casos de uso → aclarar → plan aprobado por Mohamed → implementar → verificar), en la rama `develop`.

**Auditoría previa confirmó 2 cosas que cambiaron el enfoque:**
- No había ninguna telemetría previa en el proyecto (sin Sentry/Datadog/OTel/LangSmith — búsqueda exhaustiva en todo el repo, único hit fue una dependencia transitiva en `package-lock.json`, no uso real). Sin nada que preservar ni redirigir.
- `frontend/src/lib/openai.ts` llamaba a OpenAI con `fetch()` manual, no con el SDK oficial — la instrumentación automática de Latitude (`instrumentations: { openai: OpenAI }`) solo detecta llamadas hechas a través del cliente oficial. Confirmado con Mohamed y corregido: por dentro, `openai.ts` ahora usa el SDK oficial `openai` (mismas 3 funciones exportadas — `openaiChatJSON`/`openaiChatText`/`openaiVisionJSON` —, mismo comportamiento externo, ningún otro archivo cambió su forma de llamarlas salvo el propio wrapper de captura).

**3 casos de uso reales encontrados e instrumentados** (los únicos 3 puntos de llamada a un LLM en todo el proyecto — no hay backend Python que use IA):
- **Darivo, chat de soporte** (`api/darivo/chat/route.ts`) — traza `darivo-chat`, `userId` cuando hay sesión (endpoint público), tags `["darivo","soporte"]`.
- **IA Cotización** (`api/ia/cotizacion/route.ts`) — traza `ia-cotizacion`, siempre con `userId` (ruta autenticada), tags `["ia","cotizacion"]`.
- **IA Gasto** (`api/ia/gasto/route.ts`, Cierre) — traza `ia-gasto`, siempre con `userId`, tags `["ia","gasto","vision"|"texto"]` según si viene con imagen.

**`frontend/src/lib/latitude.ts`** (nuevo): inicializa `Latitude` una sola vez a nivel de módulo (si `LATITUDE_API_KEY`/`LATITUDE_PROJECT_SLUG` no están puestas, queda en `null` y `capture()` simplemente ejecuta la función sin generar ninguna traza — mismo patrón tolerante que otras integraciones opcionales del proyecto, ej. `logo_url`). Exporta un `capture()` propio que envuelve al de la librería y llama `latitude.flush()` en cada ejecución — necesario porque las rutas de Next.js en Vercel son funciones serverless de vida corta, y sin flush explícito los spans en batch podrían perderse si la función se suspende antes de exportarlos.

**Paquetes agregados** (`frontend/package.json`): `openai@6.48.0` (SDK oficial, versión estable más reciente al momento) y `@latitude-data/telemetry@3.7.0` (última estable, no había ninguna alpha más nueva).

**Config**: `LATITUDE_API_KEY` y `LATITUDE_PROJECT_SLUG` agregadas a `frontend/.env.example` (con nota de que son opcionales — sin ellas la app funciona igual) y a la checklist de variables de Vercel del mismo archivo. `LATITUDE_PROJECT_SLUG` real ya provisto por Mohamed en el chat (no es secreto). `LATITUDE_API_KEY` — **no se pegó en el chat ni se configuró en este entorno** (regla permanente de credenciales); pendiente que Mohamed la ponga en Vercel (Production + Preview) y en su `.env.local`, igual que ya hace con `OPENAI_API_KEY`. El MCP remoto de Latitude no está disponible en el registro de conectores de este entorno — no se pudo conectar automáticamente; Mohamed puede agregarlo como servidor MCP remoto custom (`https://api.latitude.so/v1/mcp`) desde la configuración de su propio cliente si quiere que una sesión futura lo use para descubrir proyectos/keys.

**Verificado**: `tsc --noEmit`, `next lint` (mismos 3 warnings preexistentes de `useCotizacion.ts`/`useFactura.ts`, sin relación) y `next build` limpios (81 rutas, mismo conteo que antes — ningún archivo nuevo agrega ruta). **No verificado con trazas reales en Latitude** — este entorno local no tiene `OPENAI_API_KEY` ni `LATITUDE_API_KEY` configuradas (confirmado, ninguna de las 2 tiene valor en `frontend/.env.local`), así que no se pudo ejecutar ninguno de los 3 flujos reales para confirmar que las trazas llegan.
